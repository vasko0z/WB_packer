# CONTRIBUTING.md — Руководство по внесению изменений

**Версия:** 1.0
**Последнее обновление:** 2026-06-02

## Как внести вклад

1. Форкните репозиторий
2. Создайте ветку для фичи: `git checkout -b feat/название-фичи`
3. Внесите изменения
4. Убедитесь, что `SESSION_LOG.md` обновлён
5. Сделайте коммит: `git commit -m "feat: описание изменения"`
6. Запушьте: `git push origin feat/название-фичи`
7. Создайте Pull Request

## Стиль кода

- Python 3.8+
- UTF-8, LF
- Отступы: 4 пробела
- Длина строки: ≤120 символов
- Именование: `snake_case` для функций/переменных, `PascalCase` для классов

## Типы коммитов

- `feat` — новая функциональность
- `fix` — исправление бага
- `refactor` — рефакторинг без изменения поведения
- `chore` — рутинные задачи
- `docs` — документация
- `perf` — оптимизация производительности

## Тестирование

- Перед коммитом запустите приложение и проверьте основные сценарии
- Для критических изменений — сборка через `pyinstaller WB_Packer.spec --clean`

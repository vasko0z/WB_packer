# Шаблоны и правила оформления документации

**Версия:** 1.0
**Последнее обновление:** 2026-06-02

## Шаблон документа модуля/фичи

```markdown
# Название модуля/фичи

**Версия:** 1.0
**Последнее обновление:** YYYY-MM-DD

## Описание

Краткое описание назначения.

## Использование

Примеры кода и сценарии использования.

## Архитектура

Описание структуры, ключевых классов и взаимосвязей.

## Зависимости

- Зависимость 1
- Зависимость 2

## История изменений

- YYYY-MM-DD — краткое описание изменения
```

## Правила оформления

1. **Шапка:** каждый `.md` файл начинается с H1 заголовка и метаданных (версия, дата)
2. **Кодировка:** UTF-8 (без BOM)
3. **Длина строки:** ≤120 символов
4. **Даты:** ISO `YYYY-MM-DD`
5. **Версии:** semver `MAJOR.MINOR` или `MAJOR.MINOR.PATCH`
6. **Ссылки:** относительные `docs/X.md` или `<topic>.md`
7. **Имена файлов:** без эмодзи, латиница
